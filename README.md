# cc-efficiency

Detects wasteful patterns in your Claude Code usage, estimates token overhead, and gives actionable recommendations. Zero dependencies -- Python 3.8+ stdlib only.

**What it does:** Reads your local Claude Code event log, finds anti-patterns (Bash overuse, search churn, unnecessary re-reads, etc.), and tells you exactly what to change to save tokens and move faster.

**What it doesn't do:** It never reads your code, prompts, or conversation content. It only analyzes tool call events (which tool, when, which session).

## Quick Start

### 1. Install the hook (required, one-time)

The analyzer reads from `~/.claude/.dashboard-events.jsonl`. This file is generated by a PostToolUse hook. Add this to your `~/.claude/settings.json`:

```jsonc
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "node -e \"process.stdin.resume();let d='';process.stdin.on('data',c=>d+=c);process.stdin.on('end',()=>{try{const e=JSON.parse(d);const line=JSON.stringify({type:e.hook_event_name,tool:e.tool_name,sessionId:e.session_id,timestamp:Date.now()})+'\\n';require('fs').appendFileSync(require('path').join(require('os').homedir(),'.claude','.dashboard-events.jsonl'),line)}catch(err){}})\""
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "node -e \"process.stdin.resume();let d='';process.stdin.on('data',c=>d+=c);process.stdin.on('end',()=>{try{const e=JSON.parse(d);const line=JSON.stringify({type:e.hook_event_name,tool:e.tool_name,sessionId:e.session_id,timestamp:Date.now()})+'\\n';require('fs').appendFileSync(require('path').join(require('os').homedir(),'.claude','.dashboard-events.jsonl'),line)}catch(err){}})\""
          }
        ]
      }
    ]
  }
}
```

> See [`hooks.json`](hooks.json) for the full config including optional enhanced hooks (error tracking, permission denials, session tracking).

### 2. Run the analyzer

```bash
# Last 7 days (default)
python cc_efficiency.py

# Last 30 days
python cc_efficiency.py --days 30

# All time
python cc_efficiency.py --all

# Machine-readable JSON
python cc_efficiency.py --json
```

## What It Detects

| Pattern | Signal | Token Cost | Fix |
|---------|--------|------------|-----|
| **Bash Overuse** | Bash >20% of tool calls | ~400/call | Use Grep/Read/Glob instead |
| **Search Churn** | 4+ Grep/Read loops | ~1,200/loop | Use Explore agent; add paths to CLAUDE.md |
| **ToolSearch Overhead** | Frequent schema lookups | ~350/call | Pre-allow MCP tools in settings.json |
| **Read-After-Edit** | Read within 5s of Edit | ~800/call | Edit confirms success; skip re-read |
| **Rapid-Fire Edits** | 4+ Edits in 10s | ~200/edit | Batch related changes in one request |
| **Agent Overuse** | Agent >5% of tool calls | ~6,000/spawn | Use Grep/Read for simple lookups |
| **WebSearch for Local** | WebSearch then local search | ~1,500/call | Search codebase first |

## Enhanced Hooks (Optional)

For richer reports, add error and permission denial tracking. See [`hooks.json`](hooks.json) for the full config. This adds:

- **PostToolUseFailure** -- tracks tool errors (which tools fail most?)
- **PermissionDenied** -- tracks permission friction (which tools get denied?)
- **SessionStart** -- tracks session boundaries (for duration calculation)

Enhanced data is written to `~/.claude/.efficiency-events.jsonl` and appears in a separate report section.

## Claude Code Skill (Optional)

To use as a `/efficiency` slash command inside Claude Code:

```bash
# Copy the skill to your skills directory
cp -r skill ~/.claude/skills/efficiency
# Copy the script to your tools directory
mkdir -p ~/.claude/tools
cp cc_efficiency.py ~/.claude/tools/
```

Then invoke with `/efficiency` in any Claude Code session.

## How It Works

1. The PostToolUse hook writes one JSON line per tool call to `~/.claude/.dashboard-events.jsonl`
2. Each line: `{"type": "PostToolUse", "tool": "Read", "sessionId": "abc-123", "timestamp": 1713000000000}`
3. The analyzer reads this file, groups events by session, and runs 7 pattern detectors
4. Token waste estimates use conservative per-call heuristics (see `TOKEN_COSTS` in source)
5. No code, prompts, or conversation content is ever read -- only tool call metadata

## Comparison with Existing Tools

| Tool | Focus | Data Source |
|------|-------|-------------|
| `/insights` (built-in) | Qualitative AI analysis | Full transcripts |
| `/stats` (built-in) | Activity grid, streaks | Session metadata |

`ccusage` tells you "you spent $X". `cc-efficiency` tells you "you wasted 30% on search churn, here's how to fix it."
